Classical Inheritance

Javascript has really powerful prototypical inheritance system.

As powerful, in fact, as it is immensely confusing.

The last thing you want to be doing before a big deadline is trying to remember how calling your superclass works in a prototypical class structure.

so…

Wouldn’t it be nice to just write it like so?

class Actor
    constructor: (saying) ->
        @saying = saying
    say: () ->
        console.log @saying
                
class GaryBusey extends Actor 
    constructor: () ->
        super ("loose like a buse'")
gary: new GaryBusey()
this.alert gary.say()

Strong Binding

So let's just admit it: this is confusing in javascript.

Not insurmountably so; but you do have to spend a bit of mental overhead trying to determine which context your method is actually executing in.

In python, methods are strongly bound to their objects.

class AbrahamLincoln(object):
    def sword_fight(self):
        return "totally murder the other \
          dude, i mean, have you seen his limbs?"

president = AbrahamLincoln()
generic_sword_fight = president.sword_fight
generic_sword_fight.im_self is president \
  # == True! just can't forget about abe.

Let’s look at Coffeescript’s solution…

class DOMKiller 
    constructor: (domelement, message) ->
        @message = message
        $(domelement).click((event) =>
            alert @message;
        )

See! See how the fatty arrow => says “Hey buddy, I belong to the DOMKiller!”

And how the @message syntax is a shortcut for this.message? I think this a really cool addition.

Destructuring Assignments & Splats

Coffeescript brings these over, a couple of my favorite timesavers from python.

bam: () -> [0, 1]
[variable1, variable2]: bam()

you can see how it takes the two results and assigns them into variable1 and variable2.

xmen: (wolverine, cyclops, jean_grey, \
  other_less_important_xmen...) ->
    for xman in other_less_important_xmen
        alert "$xman sucks"

This is, of course, possible in javascript with the arguments super-magic keyword, but it seems a little nicer here. Also note the string interpolation with the xman variable.

Wrapping up…

There are a wealth of other features in Coffeescript. I just wanted to touch on a few of my favorites and hopefully inspire some interest in the language.

Keep in mind, of course, that it’s a new language, despite the fact that it compiles to javascript. Expect a half hour or so of frustration when you start.

To install it, just sudo brew install coffee-script on OSX.

It’s useful to run it in “watch” mode: coffee --compile --watch directory/* -o js/
